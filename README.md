# ShelfSpace
## Етап 1: Ідея та Постановка Проблеми
## Проблема
Любителям книг складно зберігати власні враження від прочитаних книг, оцінки та рецензії в одному місці. Через це відгуки губляться, а користувачам важко відстежувати прочитані книги та ділитися своєю думкою.

## Рішення
Веб-застосунок на Java для створення, перегляду, редагування та видалення рецензій на книги. Користувач може додавати книги, ставити оцінки, писати відгуки та переглядати список прочитаного.

## Цільовий користувач
Користувачі, які читають книги, ведуть список прочитаного та хочуть зберігати власні рецензії й оцінки.

## Етап 2: Проєктна Документація у README.md
## User Stories

1. Як користувач, я хочу додавати книгу та власну рецензію, щоб зберігати свої враження від прочитаного.

2. Як користувач, я хочу переглядати список своїх книг і рецензій, щоб бачити все прочитане в одному місці.

3. Як користувач, я хочу редагувати свою рецензію або оцінку книги, щоб виправляти або оновлювати інформацію.

4. Як користувач, я хочу видаляти книги зі списку, щоб прибирати непотрібні записи.

5. Як користувач, я хочу ставити оцінку книзі від 1 до 10, щоб швидко оцінювати прочитане.


## Вимірювані обмеження

1. **Performance**

   * Час відповіді API для основних операцій (додавання, перегляд списку) - не більше 2 секунд.

2. **Availability**

   * Застосунок повинен бути доступний щонайменше 95% часу під час тестування.

3. **Security**

   * Паролі користувачів повинні зберігатися у зашифрованому вигляді.
   * Користувач може редагувати та видаляти лише власні рецензії.

4. **Data Validation**

   * Оцінка книги повинна бути лише в діапазоні від 1 до 10.

## API Endpoints

### 1. Додавання книги та рецензії

### Request

`POST /api/books`

```json
{
  "title": "The Martian Chronicles",
  "author": "Ray Bradbury",
  "rating": 9,
  "review": "Книга про колонізацію Марса та людську природу."
}
```

### Response

```json
{
  "id": 1,
  "title": "The Martian Chronicles",
  "author": "Ray Bradbury",
  "rating": 9,
  "review": "Книга про колонізацію Марса та людську природу."
}
```


### 2. Отримання списку книг

### Request

`GET /api/books`

### Response

```json
[
  {
    "id": 1,
    "title": "The Martian Chronicles",
    "author": "Ray Bradbury",
    "rating": 9
  },
  {
    "id": 2,
    "title": "1984",
    "author": "George Orwell",
    "rating": 10
  }
]
```

## Test Cases

### Test Case 1 — Успішне додавання книги

**Given:** користувач відкрив застосунок

**When:** користувач додає книгу з валідними даними

**Then:** книга з’являється у списку прочитаних


### Test Case 2 — Невірна оцінка

**Given:** користувач заповнює форму додавання книги

**When:** користувач вводить оцінку 15

**Then:** система показує помилку “Rating must be between 1 and 10”


### Test Case 3 — Видалення книги

**Given:** у користувача є додана книга

**When:** користувач натискає кнопку “Delete”

**Then:** книга зникає зі списку
